Ubuntu vps设置swap解决ram不足问题

发表于 2016-02-24   |   分类于 笔记   |  

需求一直是我摸索的动力。最近在编译一些软件的时候，gcc经常报错，开始时一两次以为是配置出错导致。但是后来细心一想，可能是运存不足。用free命令查看，果不其然，256M的运存有点力不从心了。只能从有限的8G空间挤出一点来做swap解决这个问题。

查看swap使用情况

显然是不会有的。命令返回的结果是一个空表。

设定swap文件

一般情况下，创建 Swap 分区的大小都为当前系统使用的物理内存大小或内存大小的倍数。因为即使使用SSD来做，速度始终是不及内存的快，其次，部分运存会用作缓存，所以这里就建立512M就够用了。

传统方法

用dd命令来预先创建交换分区文件，然后再用/dev/zero将该文件的内容全部置零，创建时还将用到bs参数来设置要创建的交换空间大小，以及使用count来指定要创建的block数。创建交换文件的过程比较慢。

这里创建了一个512M的swap文件，文件名是swapfile，放在根目录下。

快速方法

启用swap文件

激活添加swapfile文件的权限，避免安全风险发生。这样只有root用户才能对swap有读写权限。

激活swap文件。

再次查看swap空间使用情况。成功挂载。

设置开机自动挂载swap

系统重启后是不会自动挂载刚创建的交换文件的，因此还需要修改fstab。

在文件最下方添加：

###　优化swap设置参数
毋容置疑，真正的内存是要比swap好使的，所以系统应该更加积极的使用ram，把硬件性能榨干。vm.swappiness由原来的60改成10或者激进点的0。

同样的开机自动设置，就要修改/etc/sysctl.conf。

完成之后，编译基本就不会出错了。Good！
