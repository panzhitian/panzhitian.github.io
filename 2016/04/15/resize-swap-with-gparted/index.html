用Gparted调整swap分区

发表于 2016-04-15   |   分类于 笔记   |  

上次用root分区创建了swap文件之后，无意中发现ramnode的知识库里有一项Missing Swap，里面说到SolusVM KVM模板不能在安装系统的时候正确启用256M大小的swap分区/dev/vda2。但好像只有在Ubuntu才会碰到，安装Debian和Centos都能自动挂载。现在git clone竟然也出现内存不足，只能再次调整swap分区大小。

内存不足

很久没折腾，今天想在vps上编译个android内核，但是在下载源码的时候出现错误FATA: git clone out of memory calloc failed9322，显然是内存不足。作了以下调整：

clone依然报错，下载速度飞快，但是浪费了几个小时等待处理deltas。

小插曲：单核？

其中有个小插曲是发现分配的CPU只有单核？虽然IO和网速都快的爆炸，但是觉得有点小坑啊。这个以后再研究。

调整swap

本着不浪费那256M的分区，于是决定用Gparted调整swap分区而不是创建文件。ramnode的控制面板直接提供了CDRom的Gparted live ISO，选择之后挂载。然后调整Boot Order到(1)CDROM (2)Hard Disk或者CDROM ONLY。reboot重启，然后通过vnc登陆。进到live ISO是先选keymap，看着选个English、qwerty就可以了，再是选个语言。盘头是/root分区，盘尾是swap分区。/root压缩出768M空间来，然后swap调整起始位置扩展分区。确认修改分区表，对新的swap分区做格式化，格式肯定是linux swap了，完成。将启动项改为(1)Hard Disk (2)CDROM或者Hard Disk Only即可。再次重启。

重新执行git clone

没问题，done。
